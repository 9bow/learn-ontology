## 제조업 데이터의 복잡성

현대 제조업의 데이터 환경은 금융업 못지않게 복잡합니다. 하나의 제품이 설계(CAD), 시뮬레이션(CAE), 생산 계획(ERP), 공정 제어(SCADA/DCS), 품질 검사(MES), 유지보수(CMMS) 등 수십 개 시스템을 거칩니다. 각 시스템은 서로 다른 데이터 모델과 명명 규칙을 사용합니다. "펌프(Pump)"라는 장비 하나도 설계 시스템에서는 `P-101`, SCADA에서는 `TAG_PUMP_COOLANT_1`, 유지보수 대장에서는 `냉각 펌프 #1`로 각각 등록되어 있을 수 있습니다.

Siemens와 Bosch는 이 이질성 문제를 온톨로지와 지식 그래프로 해결하려는 대표적 기업입니다.

## Siemens Industrial Knowledge Graph

Siemens AG는 2010년대 중반부터 온톨로지 기반 데이터 통합 연구를 진행해 왔습니다. ISWC(International Semantic Web Conference) 등에서 발표한 논문들을 통해 접근법의 일부를 확인할 수 있습니다.

**스트리밍 센서 데이터의 의미적 접근:** Siemens의 연구팀은 발전소와 산업 설비의 SCADA 데이터를 OWL 온톨로지에 매핑해 SPARQL로 질의할 수 있는 시스템을 구축했습니다. 이를 통해 "지난 24시간 동안 임계값을 초과한 온도 센서가 달린 터빈 컴포넌트"를 단일 쿼리로 조회할 수 있습니다.

**제품 데이터 온톨로지:** Siemens는 자사 제품 카탈로그와 기술 문서를 IEC CDD(Common Data Dictionary) 표준과 연계한 온톨로지로 구조화합니다. IEC CDD는 전기·전자 제품의 특성을 국제적으로 표준화한 데이터 사전으로, Siemens처럼 글로벌 공급망에 참여하는 기업에게 상호운용성의 기반이 됩니다.

**유지보수 예측:** 장비의 구조적 모델(어떤 컴포넌트가 어떤 서브시스템에 속하는가)을 온톨로지로 표현하고, 운영 데이터와 결합해 장애 패턴을 분석합니다. "A 타입 베어링이 장착된 모터에서 특정 진동 패턴이 나타난 후 평균 48시간 이내에 고장이 발생했다"는 패턴을 그래프 추론으로 발견합니다.

## Industrial KG 데이터 흐름

```text
제조 Industrial KG 구축 흐름
──────────────────────────────────────────────────────────
 [현장 시스템]       [온톨로지 레이어]      [KG 서비스]

 SCADA/DCS ─────┐
 (실시간 센서)   │   ┌────────────────┐
                 │   │  도메인 온톨로지│
 MES ───────────┼──▶│  IEC CDD       │──▶ 예측 유지보수
 (생산실적/품질) │   │  SSN/SOSA      │    (고장 패턴)
                 │   │  ISO 15926     │
 ERP ───────────┤   │                │──▶ 공정 최적화
 (BOM/구매)     │   │  R2RML 매핑    │    (불량률 분석)
                 │   │  레이어        │
 PLM ───────────┘   └───────┬────────┘──▶ 부품 검색
 (CAD/설계도면)              │             (IEC CDD 연계)
                             │
                    ┌────────▼────────┐
                    │  트리플 스토어  │──▶ SPARQL 질의
                    │  (GraphDB 등)   │    인터페이스
                    └─────────────────┘

 "펌프(P-101)" = "TAG_PUMP_COOLANT_1" = "냉각 펌프 #1"
  → 온톨로지로 동일 물리 자산으로 해소 (owl:sameAs)
```

## Siemens 온톨로지 스택

Siemens가 연구 논문에서 공개한 온톨로지 스택의 계층 구조입니다.

| 레이어 | 내용 | 표준/도구 |
|-------|------|---------|
| 상위 온톨로지 | 최상위 개념(물리적 객체, 프로세스, 속성) | DOLCE, BFO |
| 도메인 온톨로지 | 산업 장비, 공정, 측정 | IEC CDD, SSN/SOSA |
| 응용 온톨로지 | 특정 공장·제품 라인의 구체적 개념 | 내부 개발 |
| 인스턴스 데이터 | 실제 설비, 센서 측정값 | R2RML 매핑 |

W3C의 SSN(Semantic Sensor Network) 온톨로지와 그 핵심 모듈인 SOSA(Sensor, Observation, Sample, Actuator)는 센서 데이터를 표준화하는 데 특히 유용합니다.

## Bosch의 접근법

Bosch는 IoT 플랫폼(Bosch IoT Suite)과 시맨틱 기술을 결합하는 방향으로 산업 데이터를 관리합니다. 공개된 블로그와 연구에서 확인할 수 있는 핵심 활용 사례입니다.

**디지털 부품 패스포트:** 각 부품(나사, 센서 모듈 등)의 제조 이력, 사용된 소재, 에너지 소비량을 온톨로지로 연결합니다. 이는 EU의 디지털 제품 여권(Digital Product Passport) 규제 요건과도 맞닿아 있습니다.

**다공장 데이터 통합:** Bosch는 전 세계 수백 개 공장을 운영합니다. 공장마다 다른 MES·ERP 시스템을 사용하지만, 공통 온톨로지로 데이터를 표준화하면 공장 간 생산 효율을 비교하고 베스트 프랙티스를 전파할 수 있습니다.

**품질 데이터 분석:** 생산 공정 파라미터(온도, 압력, 가공 속도)와 품질 검사 결과를 그래프로 연결해, 어떤 공정 조건 조합이 불량률 상승과 연관되는지를 분석합니다.

## 표준 온톨로지의 채택과 확장

제조 분야에는 이미 여러 표준 온톨로지가 존재합니다.

**ISO 15926:** 석유·가스 플랜트의 생애주기 데이터 통합을 위한 표준입니다. 장비, 프로세스, 역할을 포괄하는 광범위한 상위 온톨로지(RDL, Reference Data Library)를 포함합니다.

**RAMI 4.0:** 독일 플랫폼 인더스트리 4.0이 제안한 참조 아키텍처 모델로, 자산 관리 셸(Asset Administration Shell, AAS)과 연계됩니다. AAS는 디지털 트윈의 표준 인터페이스 역할을 합니다.

**SSN/SOSA:** W3C가 권고하는 센서 데이터 온톨로지입니다. IoT 기기에서 수집한 측정값을 RDF로 표현하는 표준 방법을 제공합니다.

Siemens와 Bosch 모두 이 표준들을 기반으로 하되, 기업 내부의 구체적 요구사항을 반영한 확장 온톨로지를 추가하는 방식을 취합니다.

## 구현 시 고려사항

제조 도메인 지식 그래프 구축에서 특히 중요한 실무 고려사항입니다.

- **OT/IT 통합:** 운영 기술(OT) 시스템(SCADA, DCS)은 실시간 데이터를 생성하지만 보안 요건상 인터넷 연결이 제한됩니다. 엣지 게이트웨이를 통한 데이터 수집 아키텍처가 필요합니다.
- **레거시 시스템:** 수십 년된 PLM·ERP 시스템의 데이터를 추출해 온톨로지에 매핑하는 작업은 상당한 엔지니어링 노력이 필요합니다.
- **변경 관리:** 제품 설계가 변경되면 온톨로지 인스턴스도 함께 업데이트되어야 합니다. 버전 관리와 변경 이력 추적이 중요합니다.

## 정리

Siemens와 Bosch는 제조 도메인에서 온톨로지가 단순한 연구 주제를 넘어 실제 운영 데이터 통합에 사용되고 있음을 보여주는 대표 사례입니다. 다음 챕터에서는 이 맥락에서 가장 주목받는 개념인 디지털 트윈과 온톨로지의 관계를 살펴봅니다.
