영국 국민보건서비스(NHS)는 세계 최대 규모의 공공 의료 시스템 중 하나다. 그러나 수십 년에 걸쳐 구축된 수천 개의 이질적인 IT 시스템이 데이터를 각자의 사일로에 가두고 있었다. NHS Federated Data Platform(FDP)은 이 단편화를 해결하기 위한 국가 규모의 데이터 통합 프로젝트다.

## NHS 데이터 단편화 문제

NHS는 약 1,200개의 NHS 트러스트(병원·클리닉 운영 조직)와 7,000개 이상의 GP(일반의) 서비스로 구성된다. 각 트러스트는 독자적인 EMR 시스템(EPIC, System C, TPP SystmOne 등)을 운영한다. 그 결과:

- 환자가 다른 트러스트로 이송될 때 임상 기록을 팩스나 수동으로 전달하는 경우가 발생
- 수술 대기 명단이 트러스트별로 분산되어 전국 수준의 현황 파악 불가
- 연구·감시 목적의 데이터 통합에 수개월이 소요
- COVID-19 팬데믹 기간에 데이터 단편화가 의료 자원 배분을 방해

## FDP 아키텍처 개요

2023년 11월 NHS England는 Palantir Technologies와 5년(2023-2028) 계약을 체결했다(계약 규모는 공식 발표 기준 약 3억 3천만 파운드). NHS FDP는 각 NHS 트러스트가 데이터를 중앙에 이전하지 않고 로컬 인스턴스를 유지하면서 연합(federated) 방식으로 통합되도록 설계됐다.

**계층 구조:**

1. **로컬 FDP 인스턴스:** 개별 NHS 트러스트가 자체 Palantir Foundry 인스턴스를 운영. 데이터는 트러스트 통제하에 있다.
2. **통합 레이어:** 정해진 데이터만 국가 레이어로 집계. 집계된 데이터는 개인 식별 정보가 제거된 형태.
3. **국가 뷰:** 전국 수준의 대기 명단, 자원 현황, 공중보건 감시.

### Palantir Ontology의 역할

각 트러스트의 이질적인 데이터 모델을 통합하는 의미 계층으로 Foundry Ontology가 사용된다. 주요 Object 타입:

- `Patient`: NHS 번호(NHS Number)로 식별. 개인 식별 정보 접근은 권한 기반.
- `WaitingListEntry`: 수술·시술 대기 항목. 대기 시작일, 우선순위, 담당 트러스트.
- `Bed`: 병동 내 병상. 현황(사용 중/비어 있음), 감염 격리 상태.
- `Staff`: 임상 인력 배치 및 역량.
- `Theatre`: 수술실 예약 및 활용도.

이 Object 타입들이 SNOMED CT, LOINC 코드로 어노테이션되어 트러스트 간 동일한 의미를 보장한다.

**Actions를 통한 운영 자동화:**

수술 일정 최적화, 병상 배정, 대기 명단 관리를 Foundry Actions로 정의해 임상 직원이 UI에서 실행할 수 있다. 각 Action은 권한 검증과 감사 로그를 자동으로 기록한다.

## 실제 활용 사례

NHS England 공식 발표에 따르면 FDP는 다음 목적으로 활용된다.

**수술 대기 명단 관리:** 잉글랜드 전역의 수술 대기 환자 현황을 실시간으로 파악하고, 대기가 긴 환자를 여유 역량이 있는 트러스트로 재배치하는 의사결정을 지원한다.

**병상 관리:** 트러스트 내 병상 현황(가용, 사용 중, 청소 중, 격리)을 실시간 대시보드로 표시해 병동 관리자의 배정 의사결정을 지원한다.

**공급망 관리:** 의약품, 의료 소모품 재고를 추적하고 공급 부족을 사전에 탐지한다.

**감염 관리:** 병원 내 감염(HAI) 발생 패턴을 추적하고 위험 지역을 조기에 식별한다.

## 데이터 거버넌스와 프라이버시

FDP에서 데이터 거버넌스는 다층적으로 구성된다.

**접근 제어:** 역할 기반 접근 제어(RBAC)가 Foundry 플랫폼 레벨에서 적용된다. 임상 직원은 자신이 담당하는 환자 데이터에만 접근할 수 있다.

**로컬 데이터 통제:** 각 트러스트는 자신의 FDP 인스턴스에서 어떤 데이터를 국가 레이어로 공유할지 결정한다. 데이터 이전이 아닌 연합 접근(federated access)이 원칙이다.

**감사 로그:** 모든 데이터 접근과 Action 실행이 감사 로그에 기록된다. 로그는 변경 불가능하게 보존된다.

**식별 정보 관리:** 국가 레이어로 올라가는 데이터는 NHS 번호를 가명(pseudonymous) 식별자로 치환한다.

## 논쟁과 비판

NHS FDP는 도입 과정에서 상당한 공개 논쟁을 불러일으켰다. 균형 있는 이해를 위해 주요 비판을 함께 소개한다.

**프라이버시 우려:** Open Rights Group 등 시민단체는 대규모 민간 기업(Palantir)에 NHS 환자 데이터 인프라를 위탁하는 것에 우려를 표명했다. 특히 Palantir의 군사·정보기관 계약 이력이 논쟁의 빌미가 됐다.

**벤더 종속:** 5년 계약 이후 다른 공급자로 전환하는 비용과 복잡성에 대한 우려.

**투명성:** 계약 세부 내용과 데이터 활용 범위에 대한 충분한 공개 논의가 이루어졌는지에 대한 의문.

**NHS England의 입장:** NHS England는 환자 데이터가 NHS 통제하에 있으며 Palantir이 데이터를 상업적으로 사용하거나 제3자에게 판매할 수 없다고 공식 발표했다. 또한 독립적인 감사와 데이터 보호 영향 평가(DPIA)를 수행했다고 밝혔다.

FDP는 의료 데이터 통합의 기술적 가능성과 공공 데이터 거버넌스의 정치적 복잡성이 교차하는 사례로, 온톨로지 기반 플랫폼을 공공 의료에 적용할 때 기술 이상의 고려가 필요함을 보여준다.
