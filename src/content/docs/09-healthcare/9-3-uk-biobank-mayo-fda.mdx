대규모 의료 기관과 규제 기관은 수십 년간 축적된 임상·유전체·규제 데이터를 지식그래프로 구조화해 연구와 의사결정을 지원한다. 이 챕터에서는 세 가지 대표 사례를 살펴본다.

## UK Biobank

UK Biobank는 2006년부터 영국에서 40~69세 성인 약 50만 명의 유전체·임상·생활 습관 데이터를 수집한 대규모 바이오뱅크다. 허가된 연구자에게 데이터를 공개해 전 세계 수천 개의 연구에 활용된다.

### 데이터 구성

UK Biobank 데이터는 여러 모달리티를 포함한다.

- **유전체 데이터:** 전체 엑솜 시퀀싱(50만 명), 전장 유전체 시퀀싱(일부), SNP 어레이
- **임상 데이터:** NHS 진단 기록(ICD-10 코드), 처방 기록, 병원 입원 데이터
- **이미징:** 뇌 MRI, 심장 MRI, 복부 MRI, DXA 골밀도 측정 (약 10만 명)
- **바이오마커:** 혈액·소변 검사 수백 가지
- **자가 보고:** 생활 습관, 식이, 정신건강 설문

### 온톨로지 기반 표현형 주석

UK Biobank의 표현형(phenotype) 데이터는 Biobank Field ID 체계로 정리되어 있으며, 각 필드는 SNOMED CT, ICD-10, LOINC, HPO(Human Phenotype Ontology)로 어노테이션된다.

PheCODE 시스템은 ICD 코드 그룹을 임상적으로 의미 있는 표현형 범주로 매핑한다. UK Biobank 데이터에 PheCODE를 적용하면 GWAS(Genome-Wide Association Study)에 사용할 수 있는 이진 표현형 변수를 자동 생성할 수 있다.

### KG 활용 연구

UK Biobank 데이터에 그래프 기반 분석을 적용한 연구가 다수 발표됐다.

**다중 표현형 연구:** 유전자 변이 → 여러 질환 표현형 사이의 연관성을 KG로 시각화하고 새로운 다면발현(pleiotropy) 패턴을 발견.

**약물-유전체 상호작용:** 특정 유전자 변이를 가진 환자에서 약물 반응 차이를 KG 경로 분석으로 탐색.

**표현형 유사도 분석:** HPO 기반 환자 유사도 그래프를 구축해 희귀 질환 환자 군집을 식별.

## Mayo Clinic 임상 KG

Mayo Clinic은 대규모 임상 KG 구축과 활용에서 선도적인 의료 기관 중 하나다. 공개된 연구 문헌을 바탕으로 주요 접근법을 설명한다.

### 임상 데이터 KG 구축

Mayo Clinic은 전자의무기록(EMR)에서 구조적·비구조적 데이터를 통합해 환자 KG를 구축한다.

**구조적 데이터:** SNOMED CT 코드로 기록된 진단, LOINC 코드의 검사 결과, RxNorm 코드의 처방 데이터가 FHIR 리소스로 표준화된다.

**비구조적 데이터:** 임상 노트, 퇴원 요약, 방사선 보고서에서 NLP를 사용해 임상 엔티티를 추출하고 SNOMED CT 코드로 링킹한다. cTAKES(Clinical Text Analysis and Knowledge Extraction System) 같은 임상 NLP 도구가 활용된다.

### 약물감시(Pharmacovigilance) KG

Mayo Clinic이 공개한 연구(Ahern et al. 2020)에서 임상 KG를 약물 부작용 탐지에 활용하는 방법이 제시됐다. 약물-증상-환자 코호트를 KG로 표현하고, 특정 약물 복용 후 기대보다 높은 빈도로 나타나는 증상을 그래프 패턴 마이닝으로 탐지한다.

### 희귀 질환 진단 지원

희귀 질환은 진단까지 평균 수년이 걸리는 경우가 많다. Mayo Clinic은 환자의 증상 패턴을 HPO 용어로 표현하고, 희귀 질환 KG(OMIM, Orphanet)와 매칭해 감별 진단 후보를 순위화하는 시스템을 개발했다.

## FDA 의약품 온톨로지

미국 FDA는 의약품 규제 정보를 표준화된 형식으로 관리하기 위해 여러 온톨로지와 표준을 사용한다.

### SPL (Structured Product Labeling)

SPL은 FDA 승인 의약품의 라벨 정보(적응증, 금기, 부작용, 용량)를 HL7 CDA(Clinical Document Architecture) 기반 XML로 구조화하는 표준이다. 모든 FDA 승인 의약품의 SPL 데이터는 DailyMed에서 공개적으로 접근할 수 있다(https://dailymed.nlm.nih.gov/).

SPL에서 사용되는 코드 시스템:
- **MedDRA (Medical Dictionary for Regulatory Activities):** 부작용, 적응증 코딩. 계층적 구조(SOC → HLGT → HLT → PT → LLT).
- **NDF-RT (National Drug File — Reference Terminology):** 약물 성분, 약리 분류.
- **RxNorm:** 처방 약물의 표준 이름과 코드.

### FDA Adverse Event Reporting System (FAERS)

FAERS는 약물 부작용 자발적 보고 데이터베이스다. 보고된 부작용은 MedDRA PT(Preferred Term)로 코딩된다. 이 데이터에 KG 분석을 적용하면 약물-부작용 신호를 탐지할 수 있다.

OpenFDA API는 FAERS, SPL, 임상시험 데이터를 JSON API로 제공한다. 이를 RDF로 변환해 SPARQL 질의하는 파이프라인을 구성할 수 있다.

### WHO Drug Dictionary

WHO가 관리하는 약물 사전으로, 의약품의 성분, 브랜드명, ATC(Anatomical Therapeutic Chemical) 분류를 포함한다. 임상시험 데이터와 역학 연구에서 약물 코딩의 국제 표준으로 활용된다.

## 세 사례 비교

| 기관 | 데이터 규모 | 주요 온톨로지 | 핵심 활용 |
|------|-----------|------------|---------|
| UK Biobank | 50만 명 코호트 | ICD-10, HPO, SNOMED CT | GWAS, 표현형 연구 |
| Mayo Clinic | 수백만 환자 EMR | SNOMED CT, LOINC, HPO | 임상 의사결정, 희귀 질환 |
| FDA | 전체 승인 의약품 | MedDRA, RxNorm, NDF-RT | 약물감시, 규제 |

## 데이터 접근 및 윤리 고려 사항

의료 KG 연구에서 데이터 접근과 프라이버시 보호는 핵심 고려 사항이다.

- **UK Biobank:** 연구 신청 후 승인을 거쳐 접근. 원본 데이터는 UK Biobank 서버에서만 분석. 식별 정보 제거된 요약 통계는 더 넓게 공유.
- **Mayo Clinic:** 내부 데이터는 IRB 승인 연구에만 접근 허용. 공개 논문에서는 집계 통계만 발표.
- **FDA:** SPL, FAERS, 임상시험 데이터는 원칙적으로 공개. 단, FAERS 보고서에는 개인 식별 정보 제거.

검증 불가능한 성능 수치(예: "진단 정확도 X% 향상")는 원 논문의 방법론과 데이터 환경에 종속되므로, 이 챕터에서는 수치 대신 방법론적 접근법을 중심으로 기술했다.
