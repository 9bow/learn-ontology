## 누구를 위한 사이트인가

이 사이트는 다음 독자를 대상으로 합니다.

- 지식그래프·시맨틱 데이터 인프라를 **설계**하는 데이터/AI 엔지니어
- Palantir Foundry·AIP 또는 유사 플랫폼 **도입을 검토**하는 솔루션 아키텍트
- LLM의 환각·다중 홉 추론 문제를 KG로 **보강**하려는 AI 엔지니어
- 의료·금융·제조 도메인 온톨로지를 **다루는** 도메인 전문가와 PM

선수 지식은 가볍습니다. 관계형 데이터베이스에 대한 일반적 이해, JSON·SQL을 읽을 수 있는 수준이면 충분합니다.

## 콘텐츠 원칙

- 검증 불가능한 통계는 싣지 않습니다. 모든 인용은 stable primary source(arXiv, W3C, Wikipedia oldid, 공식 블로그)를 우선합니다.
- 대안 기술은 항상 비교 표로 제시합니다. "유일한 정답"보다 trade-off를 보여줍니다.
- 챕터당 본문 1,500자 이상을 목표로, 예시와 실제 코드/쿼리를 함께 둡니다.
