## RDFS 어휘 계층 구조

```text
RDFS 클래스 계층 (rdfs:subClassOf)
────────────────────────────────────
LivingThing
  └── Animal          (rdfs:subClassOf LivingThing)
        └── Mammal    (rdfs:subClassOf Animal)
              ├── Person
              │     └── Physician   ← :DrKim rdf:type Physician
              │                        추론: DrKim은 Person, Animal, LivingThing
              └── Bat

RDFS 속성 계층 (rdfs:subPropertyOf)
────────────────────────────────────
isRelatedTo
  └── prescribes      (rdfs:subPropertyOf isRelatedTo)
        └── prescribesAntibiotics

:DrKim prescribes :Aspirin
  → 추론: :DrKim isRelatedTo :Aspirin  (subPropertyOf 전파)

RDFS domain/range 추론
────────────────────────────────────
prescribes rdfs:domain Physician
prescribes rdfs:range  Drug

:DrKim prescribes :Aspirin
  → 추론: :DrKim rdf:type Physician  (domain으로부터)
  → 추론: :Aspirin rdf:type Drug     (range로부터)
```

## RDF만으로는 부족한 것

이전 챕터에서 배운 RDF로는 "김철수는 사람이다"를 표현할 수 있습니다. 그런데 "사람(Person)이란 무엇인가?"를 RDF만으로는 정의할 수 없습니다. "의사는 사람의 일종이다"는 어떻게 표현할까요?

이것을 위해 W3C는 **RDF Schema(RDFS)** 를 1999년 제안했습니다(2004년·2014년 개정). RDFS는 RDF에 **메타어휘(meta-vocabulary)** 를 추가합니다. 클래스를 정의하고, 클래스 간 계층을 만들고, 속성의 도메인과 범위를 지정합니다.

핵심은 RDFS도 RDF 트리플로 표현된다는 것입니다. "클래스를 정의하는 특별한 문법"이 아니라, `rdfs:` 네임스페이스의 특별한 어휘를 RDF 트리플로 사용하는 것입니다.

## RDFS 핵심 어휘

### rdfs:Class와 rdf:type

```turtle
@prefix rdf:  <http://www.w3.org/1999/02/22-rdf-syntax-ns#> .
@prefix rdfs: <http://www.w3.org/2000/01/rdf-schema#> .
@prefix :     <http://example.org/> .

# 클래스 선언: :Person은 rdfs:Class의 인스턴스
:Person rdf:type rdfs:Class .
:Animal rdf:type rdfs:Class .
:Physician rdf:type rdfs:Class .

# 개체 선언: :김철수는 :Person의 인스턴스
:김철수 rdf:type :Person .
```

`rdf:type`은 "A는 B의 인스턴스다"를 표현합니다. 줄여서 `a`로 쓸 수 있습니다.

### rdfs:subClassOf

```turtle
# "의사는 사람의 하위 클래스다" = "모든 의사는 사람이다"
:Physician rdfs:subClassOf :Person .
:Animal rdfs:subClassOf :LivingThing .
:Mammal rdfs:subClassOf :Animal .

# 추론: Mammal ⊆ Animal ⊆ LivingThing
# :포유류개체 rdf:type :Mammal → 자동으로 :Animal, :LivingThing도
```

`rdfs:subClassOf`는 **추이적(transitive)** 입니다. A ⊆ B이고 B ⊆ C이면 A ⊆ C가 자동 도출됩니다.

### rdfs:subPropertyOf

속성도 계층을 가질 수 있습니다.

```turtle
# "소유한다"는 "관련 있다"의 하위 속성
:owns rdfs:subPropertyOf :isRelatedTo .
:hasFather rdfs:subPropertyOf :hasParent .
:hasParent rdfs:subPropertyOf :hasAncestor .

# 추론: A :hasFather B → A :hasParent B, A :hasAncestor B (자동 도출)
```

### rdfs:domain과 rdfs:range

```turtle
# :prescribes의 주어는 :Physician이어야 한다
:prescribes rdfs:domain :Physician .

# :prescribes의 목적어는 :Drug이어야 한다
:prescribes rdfs:range :Drug .

# 데이터
:DrKim :prescribes :Aspirin .

# RDFS 추론 결과:
# :DrKim rdf:type :Physician  (domain으로부터)
# :Aspirin rdf:type :Drug     (range로부터)
```

**중요 — RDFS에서 domain/range는 제약이 아니다.** 데이터베이스의 외래키처럼 "이 값이 아니면 에러"가 아닙니다. RDFS 추론 엔진은 domain/range 선언을 **새 사실을 추론하는 단서**로 사용합니다. `:DrKim :prescribes :Aspirin`이 있으면, `:DrKim`이 의사라고 선언하지 않아도 추론 결과에 `:DrKim rdf:type :Physician`이 나타납니다.

## RDFS 추론 규칙 (주요 규칙)

RDFS 추론은 명확하게 정의된 규칙들로 구성됩니다. 주요 규칙을 표로 정리합니다.

| 규칙 이름 | 조건 | 결론 |
|---------|------|------|
| rdfs2 | `P rdfs:domain C`, `X P Y` | `X rdf:type C` |
| rdfs3 | `P rdfs:range C`, `X P Y` | `Y rdf:type C` |
| rdfs5 | `P rdfs:subPropertyOf Q`, `Q rdfs:subPropertyOf R` | `P rdfs:subPropertyOf R` |
| rdfs7 | `P rdfs:subPropertyOf Q`, `X P Y` | `X Q Y` |
| rdfs9 | `C rdfs:subClassOf D`, `X rdf:type C` | `X rdf:type D` |
| rdfs11 | `A rdfs:subClassOf B`, `B rdfs:subClassOf C` | `A rdfs:subClassOf C` |

이 규칙들은 **전향 연쇄(forward chaining)** 또는 **후향 연쇄(backward chaining)** 로 구현됩니다.

## RDFS 주석 어휘

추론에 참여하지 않지만 온톨로지를 문서화하는 어휘입니다.

```turtle
:Physician rdf:type rdfs:Class ;
    rdfs:label "의사"@ko ,
               "Physician"@en ;
    rdfs:comment "의료 면허를 보유하고 환자를 진료하는 사람"@ko ;
    rdfs:seeAlso <https://en.wikipedia.org/wiki/Physician> .

:prescribes rdf:type rdf:Property ;
    rdfs:label "처방하다"@ko ;
    rdfs:comment "의사가 환자에게 약물을 처방하는 관계"@ko ;
    rdfs:isDefinedBy <http://example.org/medical-ontology> .
```

## RDFS의 한계와 OWL

RDFS는 강력하지만 표현력에 제한이 있습니다. 다음 질문들에 RDFS로는 답할 수 없습니다.

1. "남성(Male)과 여성(Female)은 서로 겹치지 않는다" — DisjointWith 없음
2. "처방에는 반드시 하나 이상의 약물이 있어야 한다" — Restriction 없음
3. "면허번호는 하나여야 한다" — Cardinality 없음
4. "항생제를 처방하는 모든 의사" — 교집합·복합 클래스 없음

| 기능 | RDFS | OWL 2 |
|------|------|-------|
| 클래스 계층 | ✅ | ✅ |
| 속성 계층 | ✅ | ✅ |
| 도메인·범위 | ✅ | ✅ |
| 불교차(Disjoint) | ❌ | ✅ |
| Restriction | ❌ | ✅ |
| 카디널리티 | ❌ | ✅ |
| 역속성 | ❌ | ✅ |
| 동치 클래스 | ❌ | ✅ |
| 개체 동일성 | ❌ | ✅ (owl:sameAs) |

RDFS는 **경량 스키마**가 필요한 상황에서 빛납니다. schema.org, Dublin Core, FOAF(Friend of a Friend) 같은 광범위하게 채택된 어휘들이 RDFS 수준에서 정의되어 있습니다. 추론 복잡도가 낮아 대규모 웹 데이터 처리에 적합합니다.

OWL이 필요한 수준의 정밀도와 추론이 필요하다면, 다음 챕터에서 OWL 2 프로파일을 선택하는 기준을 배웁니다.

## 완전한 RDFS 예시

```turtle
@prefix rdf:  <http://www.w3.org/1999/02/22-rdf-syntax-ns#> .
@prefix rdfs: <http://www.w3.org/2000/01/rdf-schema#> .
@prefix xsd:  <http://www.w3.org/2001/XMLSchema#> .
@prefix med:  <http://example.org/med#> .

# ── 클래스 계층 ───────────────────────────────────
med:LivingBeing  rdf:type rdfs:Class ;
                 rdfs:label "생명체"@ko .

med:Person       rdf:type rdfs:Class ;
                 rdfs:subClassOf med:LivingBeing ;
                 rdfs:label "사람"@ko .

med:Physician    rdf:type rdfs:Class ;
                 rdfs:subClassOf med:Person ;
                 rdfs:label "의사"@ko ;
                 rdfs:comment "의료 면허를 보유한 의료인"@ko .

med:Drug         rdf:type rdfs:Class ;
                 rdfs:label "의약품"@ko .

# ── 속성 계층 ─────────────────────────────────────
med:isRelatedTo  rdf:type rdf:Property ;
                 rdfs:label "관련 있다"@ko .

med:prescribes   rdf:type rdf:Property ;
                 rdfs:subPropertyOf med:isRelatedTo ;
                 rdfs:domain med:Physician ;
                 rdfs:range  med:Drug ;
                 rdfs:label "처방하다"@ko .

# ── 데이터 (ABox) ──────────────────────────────────
med:DrKim        rdf:type med:Physician ;
                 rdfs:label "김철수 의사"@ko .

med:Aspirin      rdf:type med:Drug ;
                 rdfs:label "아스피린"@ko .

med:DrKim        med:prescribes med:Aspirin .

# ── 추론 결과 (추론 엔진이 자동 도출) ────────────────
# med:DrKim rdf:type med:Person        (subClassOf)
# med:DrKim rdf:type med:LivingBeing   (subClassOf 추이성)
# med:DrKim med:isRelatedTo med:Aspirin (subPropertyOf)
```

이 예시를 Apache Jena, RDFox, Stardog 같은 도구에 입력하면 추론 결과를 직접 확인할 수 있습니다.
