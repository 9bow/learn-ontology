## ESG 데이터의 근본적 이질성

ESG(Environmental, Social, Governance) 투자는 지난 10년간 급성장했습니다. 그러나 ESG 데이터 생태계는 심각한 이질성 문제를 안고 있습니다. MSCI, Sustainalytics, Bloomberg ESG, Refinitiv 같은 주요 데이터 제공사들은 동일한 기업에 대해 서로 다른 ESG 점수를 부여하는 경우가 많습니다. 이 불일치는 단순한 측정 오류가 아니라 **개념 정의 자체의 차이**에서 비롯됩니다.

예를 들어 "탄소 배출량"을 측정한다고 해도, Scope 1(직접 배출)만 포함하는지, Scope 2(간접 배출), Scope 3(공급망 배출)까지 포함하는지에 따라 수치가 크게 달라집니다. 또한 "이사회 다양성"을 평가할 때 여성 이사 비율만 보는 제공사도 있고, 국적·연령·전문성 다양성까지 포함하는 제공사도 있습니다.

온톨로지는 이 문제를 다음과 같이 접근합니다. **각 제공사의 측정 개념을 명시적으로 정의하고, 개념 간 관계(동의어·하위 개념·교집합)를 형식화**합니다. 그러면 사용자는 "이 데이터는 Scope 1+2만 포함한다"는 사실을 기계가 처리할 수 있는 형태로 파악할 수 있습니다.

## EU 규제 프레임워크와 온톨로지

유럽의 지속가능금융 규제는 ESG 온톨로지 설계에 강력한 구조적 요건을 부과합니다.

**EU 택소노미(EU Taxonomy):** 어떤 경제 활동이 "환경적으로 지속가능한가"를 정의하는 분류 체계입니다. 6개의 환경 목표(기후변화 완화, 기후변화 적응, 수자원, 순환경제, 오염 예방, 생물다양성)와 DNSH(Do No Significant Harm) 원칙을 포함합니다. 이 분류 체계 자체가 온톨로지적 구조를 가집니다.

**SFDR(지속가능금융공시규정):** 자산운용사가 금융 상품의 지속가능성 특성을 공시하도록 의무화합니다. Article 6(일반), Article 8(ESG 특성 홍보), Article 9(지속가능 투자 목표)로 구분됩니다. 각 분류에 해당하는 데이터 항목을 정확히 정의하고 수집하려면 명확한 개념 정의가 필요합니다.

**ESRS(유럽지속가능성보고기준):** 2024년부터 대형 기업에 적용되는 공시 기준으로, 기업이 보고해야 하는 ESG 항목을 상세히 정의합니다.

이 규제들은 자산운용사에게 "우리가 투자하는 기업이 EU 택소노미 기준을 얼마나 충족하는가"를 계산하도록 요구합니다. 이를 위해 기업 활동 데이터를 EU 택소노미 분류 체계에 매핑하는 온톨로지 레이어가 필요합니다.

## ESG 데이터 통합 흐름

```text
ESG 데이터 통합 아키텍처
──────────────────────────────────────────────────────────
 [데이터 제공사]      [표준화 레이어]      [KG 활용]

 Sustainalytics ──┐
 MSCI ESG       ──┤  ┌──────────────┐
 Bloomberg ESG  ──┼─▶│  ESG 온톨로지│──▶ 포트폴리오
 Refinitiv      ──┤  │              │    스크리닝
 기업 자체보고  ──┘  │ • EU 택소노미│
                     │   정렬 여부  │──▶ SFDR 공시
 [표준/규제]         │ • GRI 기준   │    (Art.8/9)
 EU Taxonomy ────┐   │ • TCFD 지표  │
 SFDR           ─┼──▶│ • ESRS 항목 │──▶ 공급망 ESG
 ESRS           ─┘   │              │    실사 (CSDDD)
 GRI Standards       │ Provenance:  │
                     │ 출처·시점·  │──▶ 그린워싱
                     │ 방법론 기록  │    탐지
                     └──────────────┘

 핵심 문제: 동일 기업에 대해 제공사마다 ESG 점수 상이
   → 온톨로지로 "Scope 1+2만" vs "Scope 1+2+3 포함" 명시
   → SPARQL로 동일 기준 적용 기업만 필터링 가능
```

## ESG 지식 그래프의 구조

ESG 자산관리에 특화된 지식 그래프는 일반적으로 다음 노드 유형을 포함합니다.

| 노드 유형 | 예시 | 연결 관계 |
|----------|------|---------|
| 기업(Company) | Samsung Electronics | `reportsEmission`, `hasSubsidiary` |
| ESG 지표(Indicator) | 온실가스 Scope 1 배출량 | `measuredBy`, `coversPeriod` |
| 데이터 소스(Source) | Sustainalytics, MSCI | `provides`, `usesMethodology` |
| 활동(Activity) | 반도체 제조 | `alignsWith` (EU 택소노미) |
| 목표(Target) | 2030년 탄소중립 | `committedBy`, `verifiedBy` |
| 표준(Standard) | GRI 305, TCFD | `governs`, `requiredBy` |

핵심은 **출처(Provenance) 관리**입니다. "Samsung의 2023년 Scope 1 배출량은 X톤이다"라는 트리플 하나에 대해, "이 수치는 Sustainalytics가 2024년 3월에 제공했으며, GRI 305-1 기준을 적용했다"는 메타데이터를 RDF-star나 Named Graph로 부착합니다.

## 포트폴리오 분석 쿼리 예시

ESG 지식 그래프가 구축되면 다음과 같은 포트폴리오 분석이 가능합니다.

```sparql
# EU 택소노미 기후변화 완화 목표에 적합한 매출 비중이 50% 이상인 포트폴리오 기업 조회
SELECT ?company ?alignedRevenuePct WHERE {
  ?portfolio :holds ?position .
  ?position :issuedBy ?company .
  ?company :taxonomyAlignment [
    :objective :ClimateChangeMitigation ;
    :alignedRevenuePct ?alignedRevenuePct
  ] .
  FILTER(?alignedRevenuePct >= 0.5)
}
```

```sparql
# 특정 기업의 공급망(2단계)에서 고위험 ESG 국가에 위치한 공급업체 탐지
SELECT ?supplier ?country WHERE {
  :TargetCompany :hasSupplier ?tier1 .
  ?tier1 :hasSupplier ?supplier .
  ?supplier :operatesIn ?country .
  ?country :esgRiskLevel :High .
}
```

## 데이터 통합의 실무 과제

ESG 데이터 통합에서 실무적으로 직면하는 과제들입니다.

**커버리지 불균형:** 대형 상장사는 ESG 데이터가 풍부하지만 중소기업·비상장사·신흥시장 기업은 데이터가 부족합니다. 누락 데이터를 어떻게 처리할지(추정치 사용 여부, 불확실성 표시 방법)를 온톨로지에서 명시해야 합니다.

**시간적 불일치:** 기업들은 ESG 데이터를 연 1회 보고하는 경우가 많지만 데이터 제공사마다 업데이트 주기가 다릅니다. 동일 기업에 대해 2022년 데이터와 2023년 데이터가 혼재하는 상황을 관리해야 합니다.

**재무 데이터와의 통합:** ESG 분석은 재무 데이터와 결합되어야 실질적 의미를 가집니다. FIBO의 재무 개념과 ESG 온톨로지를 연결하는 매핑이 필요합니다.

**그린워싱 탐지:** 기업이 자체 보고한 ESG 데이터와 제3자 검증 데이터, 뉴스 이벤트(예: 환경 위반 사례)를 대조해 불일치를 탐지하는 용도로도 활용됩니다.

## 기술 스택 선택

ESG 지식 그래프 구현에 자주 사용되는 기술 조합입니다.

| 구성 요소 | 선택지 | 고려 사항 |
|---------|--------|---------|
| 트리플 스토어 | GraphDB, Stardog, Neptune | SHACL 지원, 추론 성능 |
| 온톨로지 언어 | OWL 2 + SKOS + FIBO | FIBO와의 호환성 |
| 데이터 수집 | R2RML, YARRRML | 기존 데이터베이스 연동 |
| 분석 인터페이스 | SPARQL, Python (rdflib) | 분석가 친화성 |
| 시각화 | Grafana (with SPARQL plugin) | 대시보드 구성 |

## 정리

ESG 데이터 통합은 온톨로지의 핵심 가치인 **이질적 개념의 명시적 정의와 출처 추적**이 직접적으로 필요한 도메인입니다. EU 택소노미와 SFDR 같은 규제 요건은 ESG 온톨로지 설계의 방향성을 제공합니다. 금융 섹션의 마지막인 이 챕터에서 다룬 내용을 퀴즈로 확인하세요.
