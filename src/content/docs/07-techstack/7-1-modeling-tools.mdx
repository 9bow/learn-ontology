온톨로지 모델링 도구는 온톨로지 엔지니어가 클래스 계층, 프로퍼티, 제약 조건을 시각적으로 설계하고 검증하는 환경을 제공한다. 올바른 도구 선택은 팀 규모, 협업 요구 사항, 추론기 통합 필요성, 예산에 따라 달라진다.

## Protégé (스탠퍼드, 오픈소스)

Protégé는 Stanford BMIR이 개발한 오픈소스 데스크톱 OWL 편집기로, 온톨로지 커뮤니티에서 사실상의 표준으로 자리 잡았다. Java로 구현되어 있으며 OWL API를 기반으로 동작한다.

**주요 특징:**

- **OWL 2 풀 지원**: DL, EL, QL, RL 프로파일 전체를 지원한다. Manchester Syntax와 DL 구문 모두 편집 가능하다.
- **플러그인 생태계**: OWLAPI, FaCT++, HermiT, Pellet 추론기 플러그인을 설치해 온톨로지 일관성 검사와 분류를 수행할 수 있다. SPARQL 쿼리 플러그인(SPARQL Query View)도 제공된다.
- **시각화**: OntoGraf 플러그인으로 클래스 계층과 프로퍼티 관계를 그래프 형태로 탐색할 수 있다. OWLViz는 GraphViz 기반 클래스 다이어그램을 생성한다.
- **SWRL 지원**: Semantic Web Rule Language 규칙을 직접 편집하고 HermiT과 연동해 실행할 수 있다.

**한계:** 로컬 파일 기반으로 협업이 어렵다. 대형 온톨로지(클래스 수만 개 이상)에서 UI 응답 속도가 저하된다. 버전 관리는 Git 같은 외부 시스템에 의존한다.

## WebProtégé (웹 기반, 오픈소스)

WebProtégé는 Protégé의 웹 버전으로 서버에 배포하면 팀원이 브라우저에서 동시에 온톨로지를 편집할 수 있다. 변경 이력이 자동으로 기록되며 코멘트·이슈 추적 기능을 내장한다.

**협업 모델:**

- 프로젝트 단위로 온톨로지를 관리하며 사용자별 읽기/쓰기 권한을 설정한다.
- 변경 사항마다 diff가 기록되어 누가 언제 어떤 공리를 추가했는지 추적할 수 있다.
- 주석(annotation)과 토론 스레드를 클래스·프로퍼티 단위로 연결할 수 있다.

**한계:** 데스크톱 Protégé 대비 플러그인 지원이 제한된다. 추론기를 실시간으로 실행하는 기능이 없어 분류 결과를 즉시 확인하기 어렵다. 자체 서버 운영 부담이 있으며 클라우드 호스팅 옵션은 공식 제공하지 않는다.

## TopBraid Composer / EDG (상용)

TopQuadrant의 TopBraid EDG(Enterprise Data Governance)는 Protégé를 Eclipse 기반으로 확장한 상용 도구에서 출발해, 현재는 웹 기반 엔터프라이즈 거버넌스 플랫폼으로 발전했다.

**차별점:**

- **SHACL Native**: SHACL 형태 제약 조건 편집과 검증을 UI 레벨에서 완전히 지원한다. OWL 온톨로지와 SHACL 형태를 함께 관리할 수 있다.
- **SPARQL 쿼리 빌더**: 그래픽 SPARQL 편집기와 결과 테이블 뷰를 제공한다. GraphQL로 쿼리하는 인터페이스도 지원한다.
- **데이터 거버넌스 워크플로**: 온톨로지 변경에 승인 워크플로를 적용할 수 있다. 용어집, 분류 체계, 참조 데이터를 통합 관리한다.
- **SPIN/SPARQL 규칙**: SPIN 프레임워크로 OWL 대신 SPARQL로 추론 규칙을 정의할 수 있다.

**한계:** 고가의 상용 라이선스가 필요하다. 소규모 프로젝트나 연구용으로는 비용 대비 효과가 낮다.

## Stardog Studio (상용)

Stardog Studio는 Stardog 지식그래프 플랫폼과 함께 제공되는 웹 기반 IDE다. 온톨로지 편집보다 지식그래프 운영과 쿼리에 초점을 맞춘다.

**주요 기능:**

- **SPARQL IDE**: 구문 강조, 자동 완성, 쿼리 결과 테이블·그래프 시각화를 제공한다.
- **데이터 탐색기**: 트리플스토어의 리소스를 탐색하고 IRI를 클릭해 해당 엔티티의 모든 트리플을 조회할 수 있다.
- **OWL 편집**: 기본적인 클래스·프로퍼티 편집은 가능하나 Protégé 수준의 상세한 공리 편집 UI는 제공하지 않는다.
- **Reasoning 제어**: Stardog의 추론 모드(QL, RL, EL, DL, SL)를 UI에서 전환하며 쿼리 결과에 대한 추론 적용 여부를 확인할 수 있다.

**한계:** Stardog 서버 없이는 사용할 수 없는 종속 도구다. 온톨로지 설계보다 운영·질의 중심이므로 모델링 단계에서는 Protégé와 병행 사용이 일반적이다.

## 도구 비교

| 항목 | Protégé | WebProtégé | TopBraid EDG | Stardog Studio |
|------|---------|-----------|--------------|----------------|
| 라이선스 | 오픈소스 (BSD) | 오픈소스 (Apache 2) | 상용 | 상용 (Stardog 포함) |
| 배포 형태 | 데스크톱 | 웹 서버 | 웹 서버 | 웹 (Stardog 종속) |
| OWL 2 편집 | 완전 지원 | 부분 지원 | 완전 지원 | 기본 지원 |
| SHACL 편집 | 플러그인 | 미지원 | 네이티브 | 기본 지원 |
| 협업 | 미지원 | 변경 추적 | 승인 워크플로 | 팀 작업공간 |
| 추론기 통합 | HermiT/Pellet/ELK | 미지원 | 내장 | Stardog 추론 |
| 시각화 | OntoGraf, OWLViz | 기본 그래프 | 풍부한 다이어그램 | 그래프 탐색기 |
| 적합 규모 | 개인·소규모 | 소~중규모 팀 | 엔터프라이즈 | Stardog 사용 팀 |

## 도구 선택 가이드

**Protégé를 선택하는 경우:** 연구·교육 목적이거나 예산이 없을 때. OWL 공리를 세밀하게 편집해야 할 때. HermiT/ELK 추론기와 긴밀하게 통합해야 할 때.

**WebProtégé를 선택하는 경우:** 팀 협업이 필요하지만 예산이 제한될 때. 변경 이력 추적이 중요한 학술·공공 프로젝트일 때.

**TopBraid EDG를 선택하는 경우:** SHACL 기반 데이터 거버넌스를 온톨로지와 함께 운영할 때. 기업 내 다양한 부서가 용어집·분류 체계를 공동 관리해야 할 때.

**Stardog Studio를 선택하는 경우:** 이미 Stardog 트리플스토어를 운영 중이며 SPARQL 질의와 추론 모드 전환이 주요 작업일 때.
